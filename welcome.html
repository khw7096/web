"천천히 꿈을 그리자."
"미래에 이로움을 줄 수 있는것을 만들고 베풀자."

소 개

Lazypic은 콘텐츠 제작 소프트웨어를 개발합니다.
폐쇠적인 방식이 아닌, 웹 기반위에 집단지성을 발휘하여 협업합니다.
원하는 프로젝트를 자유롭게 생성하고, 공유하며 함께 이루어나갑니다.
결과보다는 진행 과정의 즐거움을 중요시합니다.
활동의 자유를 위해 Lazypic은 프리랜서 그룹 형태의 비즈니스 모델을 취하고 있습니다.
관심 분야(기초 학문, 응용기술 등)을 서로 공유하고, 함께 공부 합니다.

프로젝트 생성

스스로 프로젝트를 생성하거나 다른 사람의 프로젝트에 참여할 수 있습니다.
기업과의 계약을 통해 오픈소스 프로젝트를 생성하거나 콜라보레이션을 하는 것도 가능합니다.
여러 사람들과 프로젝트를 협업하고 싶다면 기획서를 작성하고 전파하세요!

소프트웨어 사용

상용소프트웨어 사용을 지양합니다.
오픈소스 툴을 사용하거나 필요한 툴을 만들어 사용하는 것을 지향합니다.

수익계산과 분배

수익은 프로젝트 참여자의 기여도에 따라 분배합니다.
개발의 경우 git-hours를 활용하여 수익을 계산합니다.
수익 비율을 계산하기 힘든 경우(ex.콘텐츠), 참여자끼리 회의를 통해 수익분배를 결정합니다.

메일 및 명함발급

Lazypic 전용 메일 발급을 원한다면 admin@lazypic.org 로 여러분의 Gmail을 알려주세요.
id@lazypic.org 형태의 이메일을 발급해드립니다.
명함 제작이 필요하면 admin@lazypic.org 로 요청해주세요.
명함제작에 필요한 정보: 한글명, 영문명, 주소, 전화번호, id@lazypic.org

교육문서

Lazypic 교육문서는 Github에 작성됩니다.
Lazypic의 운용 메뉴얼과 교육이 필요한 학교 및 기업의 2차 교육 자료로 활용됩니다.

개 발

현재 Lazypic 은 Go를 활용하여 많은 개발이 이루어지고 있습니다.
배포를 편하게 하기위해 실행 의존성을 줄일 방안으로 Go 언어를 활용하고 있습니다.

프로젝트를 진행할 때 해당 리포지터리에서 아래 역할을 나누고 운용합니다.
Maintainer: 리포지터리 담당 관리자
Committer: 코드를 작성, 수정, 리뷰
Contributors : 쉬운 이슈, 오타등을 처리

BSD 라이센스

대부분의 개발은 BSD 라이센스 정책을 따르며 프로젝트를 진행합니다.
여러분이 작성한 코드는 언제 어디서나 활용할 수 있습니다.
개발자가 작성한 코드는 이력과 포트폴리오, 기업 자료 등으로 자유롭게 활용 가능합니다.
또한 BSD 라이센스 특성상, 추후 클로즈드 소스로 변환하여 사용하더라도 문제없습니다.

클라우드 지원

개인, 테스트프로젝트, 기업 모두는 안정적인 소프트웨어, 서버환경을 추구합니다.
우리는 AWS 및 테스트 코드 작성을 통해서 이 부분을 해결합니다.
클라우드 자원이 필요하다면 아래 메일로 AWS 키를 요청하세요.
요청: admin@lazypic.org

Lazypic은 Amazon web service에 인프라를 구축해서 프로젝트를 진행합니다.
프로젝트가 다른 국가에서 진행되는 경우, 해당 국가 지역으로 데이터를 이동합니다.

프로젝트 종료

발전가능성이 없는 프로젝트의 경우, Legacycode 로 프로젝트를 이동합니다.
사용했던 클라우드 리소스는 제거합니다.
백업이 필요한 데이터는 AWS Glacier Deep Archive 로 보관합니다.
